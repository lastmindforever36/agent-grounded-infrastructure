# Open Source Support Pitch

AGI - Agent Grounded Infrastructure - is intended to help developers use coding
agents more safely and effectively on real projects.

## Why It Matters

Modern coding agents are capable, but long-running work often degrades because
the environment lacks durable operating structure:

- context becomes stale;
- old and new paths compete;
- subagents are used without clear scope;
- validation is weak;
- secrets and local state leak into shareable configs;
- broad goals collapse into narrow, convenient tasks.

AGI packages a practical solution:

- one router skill for broad goals;
- public-safe Codex templates;
- bounded subagent roles;
- MCP inventory and setup guidance;
- optional MCP binary installer;
- optional hooks;
- Codex + Antigravity/Gemini collaboration guidance;
- secret scanning;
- validation scripts;
- AI reviewer documentation.

## Success Criteria

This project succeeds if other developers can:

1. install the harness without exposing secrets;
2. use `$agi` as a simple entrypoint for complex work;
3. get more grounded, evidence-backed agent behavior;
4. adapt the templates to their own repos without copying private state.

## Why Support Helps

More model access allows longer validation runs, broader test matrices, and
more independent review passes on open-source agent workflows.
